# Incremental roadmap

## Phase 1 — included MVP

- Nickname and avatar color identity.
- Private invite-code rooms for 2–8 active players.
- Reconnection, spectators, lobby chat, ready state, voting, room settings, emoji reactions.
- Persistent player and match statistics in SQLite.
- Server-authoritative Tank Battle with desktop/mobile controls.
- Round scoreboard, overall party ranking, responsive UI, Docker deployment.

## Phase 2 — shared mini-game engine

Extract Tank Battle behind this adapter shape:

```ts
interface MiniGameAdapter<Input, Snapshot> {
  start(players: PartyPlayer[], settings: RoomSettings): void;
  applyInput(playerId: string, input: Input): void;
  tick(deltaSeconds: number): void;
  snapshot(): Snapshot;
  stop(): void;
}
```

Add deterministic seeded randomness, reusable countdowns, standardized scoring, replay metadata, and automated simulation tests.

## Phase 3 — mini-games

1. **Football**: circle-vs-circle impulse physics, goals, teams generated by lobby selection.
2. **Racing**: checkpoints, lap validation, boost pads, traps, anti-shortcut server checks.
3. **Chicken Hunt**: server-spawned collectibles, contested pickups, combo scoring.
4. **Survival Arena**: moving hazards, shrinking safe polygon, survival placement points.
5. **Bomb Tag**: authoritative possession transfer, immunity window, fuse countdown, explosion radius.

## Phase 4 — polish

- Web Audio sound manager and volume controls.
- Sprite/particle atlas, screen shake preferences, reduced-motion accessibility.
- Victory podium animation and achievement notifications.
- Jackbox-style room link/QR code.
- PWA installation and offline shell.
- Admin moderation, profanity filtering, and room kick/ban controls.

## Phase 5 — production scaling

- PostgreSQL adapter and migrations.
- Redis Socket.IO adapter and distributed room registry.
- Metrics, structured logs, health/readiness probes, rate limiting.
- End-to-end Playwright tests and server load tests.
- Optional WebRTC voice chat with explicit room consent.
